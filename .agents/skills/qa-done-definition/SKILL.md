# Цель

Не завершать задачу формальным "готово", пока не пройдена инженерная проверка завершенности.

# Definition of done

Проверь, что задача закрыта по следующим слоям.

## 1. Domain
- сущности определены корректно
- статусы и enum'ы согласованы
- бизнес-логика не размазана по UI
- нет явного дублирования

## 2. Database
- миграции добавлены
- foreign keys корректны
- индексы там, где нужны
- nullable поля оправданы
- rollback не сломан

## 3. Routing
- web routes добавлены
- api routes добавлены, если нужны
- route names последовательны
- middleware и auth настроены верно

## 4. Validation
- используются Form Requests там, где нужно
- нет критичных необработанных payload'ов
- сообщения об ошибках адекватны

## 5. Authorization
- policies / gates добавлены
- приватные сущности защищены
- нет доступа к чужим данным

## 6. UI / Admin
- формы usable
- таблицы readable
- статусы видны
- опасные действия подтверждаются
- нет очевидных UX-поломок

## 7. SEO
- title / description / h1 / canonical учтены, если задача о публичных страницах
- slug strategy продумана
- не создаются явные дубли

## 8. Payments / Access
- webhook flow идемпотентен
- повторная выдача доступа невозможна
- статусы заказа и платежа согласованы

## 9. Testing
Минимально проверь:
- happy path
- auth/permission path
- validation failure
- key business invariant

Если тесты не написаны, прямо скажи, какие тесты обязательны.

## 10. Deploy / Ops
- нужны ли новые env variables
- нужны ли cron entries
- нужны ли seeders
- нужны ли storage links
- нужна ли ручная миграция данных
