# Цель

Создавать публичный сайт, который:
- быстро работает на PHP shared hosting
- легко поддерживается
- удобен для SEO
- не требует отдельного frontend runtime

# Подход

- Blade для основного layout и SEO-friendly markup.
- Livewire только там, где реально нужна интерактивность.
- Tailwind для стилизации.
- Минимум клиентской сложности.
- Доступность и адаптивность по умолчанию.

# Какие страницы поддерживать

- Главная
- Обо мне
- Доула / сопровождение в родах
- Подготовка к родам
- Подготовка к партнёрским родам
- Школа материнства
- Услуги
- Цены
- Новости / статьи
- Партнёры
- FAQ
- Контакты
- Каталог курсов
- Страница курса
- Личный кабинет

# Блочная CMS

Страницы должны собираться из блоков, а не быть полностью захардкоженными.

Поддерживаемые блоки:
- Hero
- Rich text
- Feature list
- CTA
- Services grid
- Courses grid
- FAQ
- Partners grid
- Testimonials
- Pricing
- Contacts
- Custom content section

# Правила для блоков

Каждый блок должен:
- быть переиспользуемым
- иметь понятные поля
- корректно рендериться, даже если часть полей пустая
- иметь control над sort_order и publish status
- поддерживать локальную логику без захардкоженных бизнес-правил

# Livewire использовать только когда нужно

Подходит для:
- формы записи
- фильтрации новостей или курсов
- FAQ accordion с server-aware behavior
- пошаговых форм
- интерактивных CTA секций
- личного кабинета ученика

Не использовать Livewire там, где простой Blade даст тот же результат проще.

# Правила шаблонов

- Семантическая HTML-структура.
- Явные heading levels.
- Корректные meta hooks для SEO.
- Не плодить дубль-разметку.
- Локализованные тексты и labels.
- Компоненты должны быть простыми для повторного использования.

# Формы

Поддерживать формы:
- записаться на консультацию
- оставить заявку
- заказать звонок
- задать вопрос
- купить курс

Все формы должны:
- валидироваться
- иметь защиту от spam
- сохранять лид
- уметь собирать UTM/реферер, если предусмотрено
- показывать понятные сообщения пользователю

# Адаптивность

- Mobile-first мышление.
- Без горизонтального скролла.
- Кнопки и формы должны быть удобны на телефоне.
- Hero и карточки не должны ломаться при длинных заголовках.

# Результат

По задаче этого skill:
1. Определи, какие Blade views / components / Livewire components нужны.
2. Укажи, что хранится в CMS, а что допустимо захардкодить временно.
3. Реализуй шаблоны так, чтобы потом их можно было связать с API без боли.
